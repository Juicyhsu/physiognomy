路徑 B 用「CNN 直接看圖像素學習」→ 視覺能力強，但難以解釋原因
兩者的結論交給 LLM → 跨路徑整合，生成個性化完整報告4.2　路徑 A：幾何規則引擎輸入：MediaPipe 輸出的 468 個坐標點計算：臉部寬高比、三庭均衡度（標準差）、印堂寬度比、眉壓眼指數、人中長度比等 17 個幾何特徵判斷：if-else 規則引擎，例如「眼距佔臉寬 &gt; 0.32 → 判定大眼」優點：每個結論都能說出原因，完全透明可解釋4.3　路徑 B：CNN 深度學習分類輸入：64×64 像素的五官 ROI 圖片（= 12,288 個數字）模型：MobileNetV2（遷移學習），為每個部位訓練一個獨立分類器輸出：分類標籤 + 信心度（例如：挺鼻型 87%、標準鼻型 10%、寬鼻型 3%）4.4　遷移學習詳解MobileNetV2 預訓練底子：在 ImageNet 百萬張圖上訓練，已學會邊緣、紋理、形狀等基礎視覺知識凍結前 80% 的層：保留 ImageNet 學到的通用視覺知識，訓練時不更新這些參數解凍後 20% 的層：讓模型做 Domain Adaptation，學習五官分類這個新任務新增分類頭：Dense(256) → Dropout(0.4) → Dense(128) → Dropout(0.3) → Softmax4.5　LLM 融合推理層支援的 LLM：Google Gemini 3 Flash Preview（主力）、Anthropic Claude、OpenAI GPT-4oLLM 的優勢：能跨部位整合考量，根據信心度、幾何數值、其他部位組合，生成個性化報告Prompt 結構：角色設定 + 幾何特徵（17 個數值）+ CNN 分類結果（5 個部位）+ 輸出格式指定（古籍格局/性格天賦/流年運勢 + JSON 評分）第五章　實驗設計與方法5.1　訓練策略概覽Batch（批次）：每次拿去訓練的一小包圖片，設定 32 張/批（Exp D 為 16 張）1 個 Epoch：把全部 161,652 張訓練圖看完一遍 = 約 5,052 次參數更新，每 Epoch 約 530 秒訓練上限：3 類任務設 30 Epoch，臉型 4 類設 35 Epoch，早停機制通常提前終止5.2　優化策略早停機制（EarlyStopping）——patience = 6白話說明連續 6 個 Epoch 驗證準確率都沒有提升，就自動停止，並把模型回滾到歷史最佳版本（restore_best_weights=True）。
就像烤蛋糕：「如果連看 6 次都還沒熟，就取出先前最熟的版本，不再繼續烤了。」學習率自動衰減（ReduceLROnPlateau）——patience = 3，factor = 0.5白話說明連續 3 個 Epoch 驗證 Loss 都沒有下降，就把學習率砍半，用更小的步伐繼續找最低點。
patience=3 &lt; patience=6 的設計邏輯：先調速（第 3 輪縮步伐），再決定停（第 6 輪才停），有先後順序。5.3　四組對照實驗設計每次只改一個超參數，其他保持不變——科學實驗的基本原則：實驗定位骨幹網路學習率批次Epoch上限改變的變因Exp A基準對照組MobileNetV21e-43230（基準，無改變）Exp B骨幹架構對比EfficientNetB01e-43230換骨幹網路Exp C低學習率組MobileNetV25e-5 ↓3235學習率減半Exp D小批次組MobileNetV21e-416 ↓30批次大小減半第六章　實驗結果6.1　四組實驗各部位最高驗證準確率部位Exp A 基準Exp B 換骨幹Exp C 低LRExp D 小批次最終採用最高值眼型71.73% ★71.72%71.73%71.72%Exp A71.73%嘴型68.22% ★63.24%49.78%42.50%Exp A68.22%眉型62.51%62.51%63.09%63.26% ★Exp D63.26%鼻型53.04%52.22%53.02%55.01% ★Exp D55.01%臉型42.48%41.49%53.22% ★41.49%Exp C53.22%圖1. 眼型四組實驗驗證準確率比較6.2　三種基準比較（多數類別基準）本報告採用兩種基準進行比較，多數類別基準比均勻隨機更嚴格，更能反映模型的真實學習能力：部位均勻隨機基準多數類別基準CNN 最終超越均勻猜超越多數類診斷說明眼型33.3%70.5%71.7%+38.4%+1.2%⚠️ 幾乎沒超越多數類模型偏向猜明亮眼型眉型33.3%62.8%63.3%+30.0%+0.5%⚠️ 最嚴重幾乎等於懶人模型鼻型33.3%55.4%55.0%+21.7%−0.4%❌ 低於多數類基準未有效學習少數類嘴型33.3%39.3%68.2%+34.9%+28.9%✅ 真正有效三類分布均勻臉型25.0%35.8%53.2%+28.2%+17.4%✅ 有效低LR策略突破核心結論嘴型（+28.9%）和臉型（+17.4%）在多數類別基準上有顯著超越，是本系統真正有效的分類器。

如果模型每次都猜「普通學生」，準確率高達 97%，但完全沒學會識別天才。
本系統眼型即為此情況：明亮眼型佔 70.5%，模型傾向大量預測「明亮眼型」，
表面上 71.7% 的準確率，實際對細眼型和疲態眼型的識別能力很可能很差。本系統的類別不平衡問題CNN 未設定 class_weight：訓練時對少數類別沒有額外重視，直接受類別不平衡影響RF 已設定 class_weight='balanced'：兩者在不平衡處理上條件不對等最嚴重部位：眼型（明亮眼型 70.5%）和眉型（標準眉 62.8%）三種解決方法方法做法優點缺點① Class Weight對少數類別的損失乘以較大的權重一行程式碼即可，不需改資料集極度不平衡時效果有限② 過採樣（SMOTE）對少數類別插值，人工生成更多樣本直接增加少數類別樣本數插值可能產生不自然樣本③ Focal Loss（推薦）動態調整損失，讓模型專注於難分類樣本不需改資料集，自動處理，目前最主流需調整 gamma 參數Focal Loss 來自 Facebook AI Research 的 RetinaNet 論文（Lin et al. 2017），是目前深度學習處理類別不平衡最被廣泛採用的方法，建議作為本系統後續改進的優先方向。7.3　研究限制限制一：CelebA 標籤的主觀性標籤由人工標注員判斷，「大鼻型」「濃眉」的標準因人而異，是資料來源的根本性限制。限制二：類別設計受限於 CelebA傳統面相學的「懸膽鼻」「鷹鉤鼻」「桃花眼」等細分類別，因 CelebA 無此標籤而無法實現。細粒度解讀由 LLM 層承接。限制三：笑容對嘴型分類的干擾當受測者在照片中有明顯笑容時，嘴型分類容易被誤判為「笑容型」，即使本人的唇型並非豐唇或特殊類型。第八章　最終模型確立與 Ensemble 策略8.1　明星隊 Ensemble 策略部位採用實驗最終 val 準確率比較 RF採用原因眼型Exp A71.73%+3.0%第 1 個 Epoch 即達最高點，遷移特徵匹配度高嘴型Exp A68.22%+8.3%低學習率反而拉低至 49%，標準設定最優眉型Exp D（Batch=16）63.26%+6.9%小批次噪聲有助跳出細毛流的局部極值鼻型Exp D（Batch=16）55.01%+11.4%鼻翼細節在小批次下泛化效果更好臉型Exp C（低學習率）53.22%+9.4%低學習率保護遷移特徵，對細微 4 類邊界最有效沒有任何單一超參數設定能在所有五官任務上都達到最優——這本身就是一個重要的實驗發現，也是 Ensemble 策略合理性的直接依據。第九章　大型語言模型整合與議題討論9.1　LLM 在本系統的角色跨路徑整合：同時考量幾何數值與 CNN 分類，二者互補跨部位考量：「挺鼻 + 印堂開闊 + 上揚眉」的組合解讀，比各部位獨立解讀更準確個性化生成：即使都是挺鼻型，87% 信心度和 52% 信心度的解讀語氣不同9.2　是否應加入 LLM 視覺解讀？面向本系統（CNN 特徵 → LLM 文字）替代方案（直接傳照片給 LLM）準確率有明確量化指標（71.73% 等），可驗證無法提供可量化的分類精度可解釋性CNN 信心度可解釋每個結論的依據LLM 直接看圖的推理過程是黑盒子成本CNN 本地推理幾乎零成本每次需上傳高解析度圖片，API 費用大幅增加學術深度有完整訓練實驗設計與對照比較，學術價值高直接使用現有 API，缺少模型訓練的學術深度隱私人臉照片只在本地推理，不需上傳外部人臉照片必須上傳至 OpenAI 或 Google 伺服器設計結論本系統選擇「CNN 特徵提取 → LLM 文字融合」架構的四個原因：

中期：表情抗擾機制 + 細粒度標籤設計
長期：MobileViT 骨幹升級 + WebRTC 即時視訊 + 用戶回饋機制
